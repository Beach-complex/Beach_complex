## 문제 상황

### 증상
- test.md 이슈 템플릿이 정상적으로 작동하지 않음
- 파일 내용은 정상처럼 보이지만, 저장/커밋 이후부터 문제 재현

### 발생 일시
- 2026-01-06 UTF-8 BOM이 포함된 파일이 커밋/머지된 이후

### 영향 범위

## 충돌 원인 분석

### 핵심 원인: UTF-8 with BOM
- 문제 파일이 **UTF-8 (without BOM)** 이 아니라 **UTF-8 with BOM** 으로 저장되어 있었음
- UTF-8 BOM은 파일 시작 부분에 붙는 바이트로, 일부 도구/파서/컴파일러는 이를 “문자”로 해석하여 오류를 발생시키거나,
  Git diff/merge에서 “첫 줄이 바뀐 것”처럼 인식되어 충돌을 유발할 수 있음

### 관측된 특징(전형 패턴)
- 파일 내용을 육안으로 봤을 때는 문제를 찾기 어려움(보이지 않는 문자)

### 확인 방법(vscode 기준)
- VSCODE에서 인코딩 확인을 통해 BOM 포함 여부를 확인
  - BOM이 있는 파일은 “UTF-8 with BOM”으로 표시됨
- 동일 파일이라도 편집기/OS에 따라 저장 방식이 달라져 BOM이 생기거나 사라질 수 있음

## 해결 과정

### 1) 문제 파일 식별
- 이슈 템플릿에 모든 파일에 인코딩 및 양식 비교

### 2) BOM 제거
- vscode에서 UTF-8로 변경

### 3) 정상 동작 검증
- 인코딩이 제대로 처리 되었는지 확인

### 4) (선택) CI/훅으로 사전 차단
- 향후 재발 방지를 위해 BOM 탐지 스크립트를 CI 또는 pre-commit에 추가
