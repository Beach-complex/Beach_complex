# 기술 결정 기록

## 1. 목적
- 이 프로젝트에서 사용하기로 한 주요 기술/라이브러리/아키텍처 결정과 그 이유를 기록합니다.
- 나중에 새로 합류한 사람이 "왜 이걸 쓰고 있지?"라는 질문에, 이 문서를 보고 이해할 수 있도록 합니다.

## 2. 기록 대상
- 프레임워크 선택 (예: Spring Boot, Express 등)
- 데이터베이스 / 캐시 / 메시지 브로커 선택
- 인증/인가 방식 (JWT, 세션, OAuth 등)
- 로깅 / 모니터링 / 배포 방식 등

## 4. 기술 결정 문서 템플릿

새로운 기술 결정을 할 때는 아래 템플릿으로 새 페이지(예: `ADR-00X-주제`)를 추가합니다.

~~~markdown
# ADR 번호: [제목]

## 상태

[Proposed | Accepted | Deprecated | Superseded]

## 컨텍스트
이 결정이 필요한 배경과 관련된 정보를 서술합니다.
현재 시스템 구조나 기술 상황, 외부 제약 조건 등을 기술합니다.

## 결정
어떤 결정을 내렸는지를 명확하게 서술합니다.
선택한 아키텍처나 도구, 패턴, 구조 등과 그 이유를 기술합니다.

## 근거
왜 이 결정을 내렸는지 설명합니다.
다른 대안들과 비교한 장단점, 주요 고려 요소 등을 포함합니다.

## 결과
이 결정이 시스템에 미치는 영향이나, 향후 따라야 할 구현 및 운영 지침을 기술합니다.
추가로 생기는 작업이나 영향 범위 등도 포함됩니다.

## 대안
검토했던 대안들과 선택하지 않은 이유를 요약합니다.
