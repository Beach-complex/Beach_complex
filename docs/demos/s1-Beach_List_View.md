## 데모 시나리오 1 해수욕장 리스트 조회(list)
### 목표:
사용자가 지역/조건 기반으로 해수욕장 시트를 조회하고, 각 항목에서 현재 혼잡도/날씨 요약을 빠르게 파악한다.

### 사전 조건:
- 리스트가 정상 동작하며, 결과에 해수욕장 기본 정보 + 혼잡도/날씨(날씨와 온도 정도)/즐겨찾기기가 포함되어야 한다.
- 태그와 검색이 동작해야 한다.


### 데모 스크립트(사용자 액션)
1. 로그인 완료 후 해수욕장 리스트 페이지로 이동합니다.
2. 태그 또는 검색어로 결과를 좁힌다.
3. 리스트에서는 다음을 보여준다.
    - 해수욕장명
    - 현재 혼잡도 상태(여유/보통/혼잡 등)
    - 날씨 요약(기온/현재 날씨)
    - 즐겨찾기

참고 : 동일 조건으로 재조회 시 응답이 빠르게 유지되는 것을 보여주면 좋음(redis)`

### 기대 결과 :
리스트가 정상 랜더링 되고, 각 카드 행에 혼잡도/날씨 요약/즐겨찾기가 표시된다.
태그/검색 변경 시 결과가 즉시 반영된다.

### 성공기준
사용자가 리스트에서 태그/검색어를 바꿔도 혼잡도/날씨/즐겨찾기가 포함된 해수욕장 목록이 빠르게 갱신된다.
