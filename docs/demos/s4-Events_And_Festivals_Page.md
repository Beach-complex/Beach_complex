## 시나리오 4 행사 축제 페이지:
### 목표 :
사용자가 행사/축제 전용 페이지에서 지역/기간 기반으로 이벤트를 탐색하고, 연관된 해수욕장을 탐색 및 예약을 할 수 있게 한다.

### 사전 조건
- 행사/축제 데이터 최소 3개 이상
  - 예정 1개 이상 
  - 진행 중 또는 종료 1개 이상
- 이벤트가 특정 해수욕장과 연결되어 있다면
  - 이벤트 상세에서 관련 해수욕장 보기 또는 예약하기로 연결 가능해야 함

### 데모 스크립트
1. 행사/축제 페이지로 진입
2. 행사 리스트에서 다음 정보가 카드/행 형태로 표시되는지 확인
    - 행사명
    - 일정
    - 위치
    - 상태(예정/진행/종료)
3. 행사 1개 클릭 -> 행사에 대한 정보를 제공
4. 행사에 대한 정보는 다음이 있어야 한다.
    - 상세 설명
    - 일정/장소
    - 관련 해수욕장 정보가 있다면 노출
5. 연결 흐름
   - 관련 해수욕장 상세로 이동 -> 예약하기로 이동
   - 또는 행사 상세에서 바로 예약

### 기대 결과
행사/축제 전용페이지에서 이벤트 목록이 안정적으로 렌더링된다.
행사를 클릭시 핵심 정보(일정/장소/설명)가 누락없이 노출된다.
행사 상세 정보도에서 해수욕장 상세/예약으로 이어지는 링크가 동작한다.

### 성공기준
사용자가 행사/축제 페이지에서 이벤트를 탐색하고 상세를 확인한 뒤, 관련 해수욕장상세 또는 예약 플로우로 끊김 없이 이동할 수 있어야 한다.
